# Work CLI QA Work Type

Use this workspace for one smoke-test scenario or debug loop.

Required lifecycle:

1. State the scenario being tested.
2. Record the command or harness step.
3. Capture pass/fail evidence.
4. If it fails, record the fix loop: improve, retest, verify fixed.

Keep artifacts small. Large logs belong in an external artifact directory and
should be linked from the work item metadata or report.
